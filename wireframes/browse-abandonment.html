Implementation notes (for the Klaviyo build)

  • E1 is reused, not rebuilt. Everything after the split is the new work.
  • Entry filter change: drop the "Placed Order = 0 all time" gate so customers can enter; the buyer/non-buyer decision moves to the split (Placed Order all-time = 0 → Prospect).
  • Prospect offer stays modest — standing 10% (WELCOME10). Browse is a weaker signal than an abandoned checkout, so no deeper discount.
  • Customer lane runs on value, not price — generic "completes your routine" + proof framing, protecting margin on people who already buy.
  • Copy stays generic (important): the only dynamic element is the viewed-product block. We can't reference the customer's past purchase yet — per-product / per-category emails are a later phase.
  • All gates on Placed Order. Exit if they place an order or start a checkout (checkout intent hands off to the AC flow, no double-send).
  • 2 SMS total, one closing each lane, consented only, GSM-7 safe.
  • Cadence: Day 0 (+2h) → Day 1 → Day 2. Short window; browse intent decays fast.
  • Dynamic viewed-product block in every message.
